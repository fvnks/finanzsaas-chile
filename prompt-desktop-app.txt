Tengo una aplicación web SaaS de gestión empresarial (ERP/finanzas) con base de datos PostgreSQL.
El schema ya existe y está en producción.

Quiero crear una aplicación de escritorio NATIVA que se conecte DIRECTAMENTE a esa base de datos PostgreSQL
sin ningún servidor HTTP intermedio (sin Express, sin API REST, sin navegador embebido).

La app debe leer y escribir datos mediante queries directas a PostgreSQL, y mostrar la información
al usuario a través de una UI nativa de escritorio.

=============================================================
STACK TECNOLÓGICO
=============================================================

- Runtime:     .NET 8 (C#)
- UI:          Avalonia UI 11 (cross-platform: Windows, macOS, Linux)
- ORM:         Entity Framework Core 8 + Npgsql provider (PostgreSQL)
- Patrón:      MVVM (Model - View - ViewModel)
- Auth local:  BCrypt para verificación de contraseñas (ya almacenadas en la DB)
- Reportes:    QuestPDF o similar para generación de PDFs nativos
- Gráficos:    LiveCharts2 para Avalonia (equivalente a Recharts)

=============================================================
ARQUITECTURA
=============================================================

FinanzSaaS.Desktop/
  ├── Models/               # Clases C# que mapean las tablas PostgreSQL (igual al schema Prisma)
  ├── Data/
  │   ├── AppDbContext.cs   # EF Core DbContext con todas las entidades
  │   └── DbConfig.cs       # Lectura/escritura de configuración de conexión
  ├── ViewModels/           # Un ViewModel por módulo (lógica de negocio, queries)
  ├── Views/                # Vistas Avalonia XAML (UI nativa)
  │   ├── MainWindow.axaml
  │   ├── LoginView.axaml
  │   ├── DbSetupView.axaml # Pantalla de configuración de DB
  │   └── [un archivo por módulo]
  ├── Services/             # Servicios de datos (repositorios)
  ├── Converters/           # Value converters para binding XAML
  └── App.axaml             # Entry point Avalonia

CONSIDERACIONES ADICIONALES
=============================================================

- No hay servidor HTTP. No hay API REST. No hay Express. No hay Node.js.
- La app es 100% nativa: ejecutable .NET que habla directamente con PostgreSQL via Npgsql.
- El PostgreSQL puede ser local o remoto (Railway, Supabase, etc.) — solo importa el connection string.
- Los arrays PostgreSQL (String[], p.ej. modules[], allowedSections[]) se mapean con EF Core
  usando el proveedor Npgsql que los soporta nativamente.
- Las contraseñas en la tabla User están hashed con BCrypt — usar BCrypt.Net para verificarlas.
- Los UUIDs se mapean como Guid en C#.
